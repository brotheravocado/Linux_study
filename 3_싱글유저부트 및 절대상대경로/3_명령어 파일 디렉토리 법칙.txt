** 명령어 법칙**
1. 명령어 [option] [target]
	ls -l /
2. 명령어의 옵션은 순서없이 조합이 가능
	ls -alR = ls -Ral
3. 띄워쓰기시 새로운 옵션 및 경로 적용
	mkdir /anon1 /anon2 /anon3 /anon4
4. 모순되는 옵션존재시 마지막 옵션이 적용
	ls -hhl
5. 명령어 Error 났을때도 띄워쓰기 이후는 무조건 실행
---------------------------------------------------
** 파일 및 디렉토리 법칙 **
1. 파일과 디렉토리 이름에는 / 를 사용할 수 없다. /는 경로명에서 구분자로 사용하기 때문이다
2. 파일과 디렉토리 이름에는 알파벳(대소문자),숫자,붙임표(-),밑줄(_),점(.)만 사용한다
	suzi.txt, hello.c, optimus, prime11
3. 디렉토리 이름에는 공백 문자,|,",',#,$,& 등을 사용하면 안된다
	&kgitbank, *windows, love suzi, iu|
4. 파일과 디렉토리 이름의 영문자는 대문자와 소문자로 구별하여 다른 글자로 취급한다
   = 미국정보교환표준부호 = ASCII(American Standard Code for Information Interchange)
5. 파일과 디렉토리 이름이 . 으로 시작하면 숨김 파일로 간주한다 ≠ . 혼자 쓰이면 현재를 의미한다

touch test/  => 실행안됨
touch = 파일 생성 명령어
test/ => 파일이름뒤에 붙는 / 은 디렉토리를 의미
